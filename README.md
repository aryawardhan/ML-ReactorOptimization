## Project Overview

In chemical manufacturing, optimizing reactor conditions directly impacts **product yield and profitability**. This project simulates a real-world CSTR environment where:

- Process variables (Temperature, Pressure, Catalyst Loading, Residence Time) influence chemical yield
- A physics-grounded kinetic model (Arrhenius equation) governs the underlying data
- Multiple ML models are trained and compared to predict yield accurately
- SHAP values explain **why** the model makes each prediction (black-box → interpretable)
- Economic optimization finds the **most profitable operating conditions**

This project demonstrates the intersection of **Chemical Engineering domain knowledge** and **Data Science/ML** — a key skill in modern process industries.

## Module Breakdown

### `01_generate_data.py` — Synthetic Data Generation
- Generates **1,500 experimental runs** with randomized process variables
- Uses **Arrhenius kinetics** (`k = exp(-Ea/RT) × 150`) for physics-grounded yield calculation
- Introduces non-linear effects: thermal degradation peak at 180°C, catalyst saturation at 1.5 mol%
- Adds realistic Gaussian noise (σ = 3%) to simulate experimental error
- Output: `data/reaction_data.csv`

**Process Variables:**

| Variable | Range | Unit |
|---|---|---|
| Temperature | 120 – 220 | °C |
| Pressure | 10 – 50 | bar |
| Catalyst Loading | 0.1 – 2.5 | mol% |
| Residence Time | 0.5 – 6.0 | hours |

---

### `02_train_models.py` — Model Training & Comparison
- Injects 5% missing values in Temperature to simulate real-world data quality issues
- Applies **Median Imputation** → **StandardScaler** → **Model Training** pipeline
- Trains and compares 3 models:

| Model | RMSE (%) | R² |
|---|---|---|
| Linear Regression | ~6.5 | ~0.65 |
| Random Forest | ~4.0 | ~0.88 |
| **XGBoost** | **~3.5** | **~0.92** |

- Saves best model + full pipeline artifacts to `models/`

---

### `03_explain_physics.py` — SHAP Explainability
- Uses **SHAP TreeExplainer** on the XGBoost model
- Generates 3 plots saved to `visuals/`:
  - **Summary plot** — impact of all features across all predictions
  - **Dependence plot** — how Temperature alone drives yield changes
  - **Bar chart** — ranked feature importance

SHAP makes the ML model interpretable for chemical engineers — showing *which process variable matters most and why*.

---

### `04_optimize_economics.py` — Economic Optimization
- Defines a profit function: `Profit = (Yield × Product Value) - (Temp Cost + Catalyst Cost)`
- Uses **Scipy L-BFGS-B** optimizer with physical bounds on all variables
- Finds the operating point that **maximizes profit per cycle**
- Saves results to `models/optimal_conditions.json`

**Economic Parameters:**

| Parameter | Value |
|---|---|
| Product Value | $500 / % yield |
| Temperature Cost | $2.5 / °C |
| Catalyst Cost | $150 / mol% |

## SHAP Physics Explanations

SHAP (SHapley Additive exPlanations) reveals the physics captured by the ML model:

- **Residence Time** has the highest positive impact — longer reaction time increases yield
- **Temperature** shows a non-linear effect — too high causes thermal degradation (peak at 180°C)
- **Catalyst Loading** shows saturation behavior — beyond 1.5 mol%, additional catalyst hurts yield
- **Pressure** has a moderate positive linear effect

These insights align with the underlying Arrhenius kinetics — validating that the model learned real chemical physics, not just noise.

---

## Economic Optimization

The optimizer balances yield maximization against operating costs:

```
Optimal Conditions:
  Temperature     →  160.0 °C
  Pressure        →  30.0 bar
  Catalyst Load   →  1.19 mol%
  Residence Time  →  3.0 hours
  Predicted Yield →  24.69%
```

The result shows that maximum yield (highest temperature, most catalyst) is NOT the most profitable — operating costs must be balanced against yield gains.
